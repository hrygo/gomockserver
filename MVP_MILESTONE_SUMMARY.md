### 设计模式应用

1. **适配器模式** (Adapter Pattern)
   - `HTTPAdapter` 统一不同协议的请求/响应模型
   - 便于未来扩展 WebSocket、gRPC 等协议

2. **策略模式** (Strategy Pattern)
   - 规则匹配支持多种策略（Simple、Regex、Script）
   - 响应生成支持多种类型（Static、Dynamic、Proxy）

3. **仓库模式** (Repository Pattern)
   - 数据访问层抽象，隔离业务逻辑和数据存储
   - 便于测试和数据源切换

4. **依赖注入** (Dependency Injection)
   - 构造函数注入依赖
   - 提高可测试性和模块解耦

### 核心算法

#### 规则匹配算法
```
1. 按项目 ID 和环境 ID 过滤规则
2. 过滤仅启用的规则
3. 按优先级降序排序
4. 遍历规则列表：
   a. 检查协议类型
   b. 匹配 HTTP 方法
   c. 匹配路径（支持路径参数）
   d. 匹配 Query 参数
   e. 匹配 Headers
   f. 检查 IP 白名单
5. 返回第一个完全匹配的规则
```

**时间复杂度**: O(n)，n 为规则数量  
**空间复杂度**: O(1)  
**优化**: 通过优先级排序和短路评估，平均情况下匹配效率更高

   - **学习**: MongoDB 驱动细节需要注意
